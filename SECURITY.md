## Reporting a Vulnerability

Please **do not** open a public issue for security-sensitive reports.

Instead, use one of the following:

1. **GitHub**: Open a [private security advisory](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) for this repository (Security tab → Report a vulnerability), if that feature is enabled.
2. **Maintainer contact**: If you maintain a fork or need another channel, contact the repository owners through the email or process listed in the repository settings or organization profile.

Include enough detail to reproduce or understand the issue (affected component, steps, impact). We will treat reports as confidential. Given the as-is nature of this project, we cannot commit to a response timeline or guarantee that a fix will be issued.
